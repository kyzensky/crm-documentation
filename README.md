# ТЗ CRM MVP

## Права доступа

### Менеджер

Менеджер имеет доступ только к своим заявкам.

### Координатор заявок

Координатор имеет доступ ко всем заявкам.

## Разделы

### Заявки

Доступные действия:
- просмотр списка всех заявок;
- просмотр карточки заявки;
- редактирование информации по заявке;
- изменение статуса заявки.

### Отчеты

Доступные действия:
- просмотр списка отчётов;
- создание отчетов
- редактирование отчетов

### Отчет менеджера

Доступные действия:
- просмотр отчетности по работе менеджера;
- добавление отчетности менеджера

## Этап 1. Создание заявки

**Роль:** Менеджер

Менеджер создаёт заявку, выбирает компанию и вносит основную информацию.

На этапе создания заявки менеджер также заполняет финансовые данные:
- общая сумма;
- расход (если есть);
- НДС;
- бонус (если есть).
- план

### Автоматическая логика после сохранения
- Система присваивает заявке статус **«Не оплачено»**.
- Система автоматически назначает ответственным **координатора заявок**.
- Система отправляет **email-уведомление координатору**.

---

## Этап 2. Работа координатора

**Роль:** Координатор заявок

На данном этапе заявка находится в зоне ответственности координатора.

Координатор выполняет следующие действия:
- заходит в банковские системы (веб-сайты / приложения банков);
- получает отчётность за предыдущий день;
- сверяет поступившие оплаты по банковской отчётности с заявками, находящимися у него в работе;
- определяет, по каким заявкам была совершена оплата;
- при подтверждении факта оплаты вручную проставляет в заявке признак **«Оплачена»**.

### Автоматическая логика после сохранения
- перевести заявку в статус **«Оплачено»**;
- отправить **email-уведомление менеджеру**;
- назначить заявку обратно **менеджеру**.

### Ограничение доступа
Платёжная информация и внутренние комментарии координатора **не должны отображаться менеджеру после возврата заявки**.
